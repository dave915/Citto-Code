# 08. 파일/모델/오류 처리

## 1) 파일 첨부
- 시스템 파일 선택 다이얼로그로 멀티 선택
- 읽기 성공 파일만 첨부 목록 반영
- 첨부 목록은 경로 기준 중복 제거
- 개별 첨부 제거 버튼 제공

## 2) 프롬프트 조립 규칙
첨부 파일이 있으면 아래 포맷으로 프롬프트 앞에 삽입:

```txt
<file path="ABS_PATH">
FILE_CONTENT
</file>
```

- 파일이 여러 개면 빈 줄로 구분
- 사용자 텍스트가 있으면 파일 블록 뒤에 2줄 개행 후 텍스트 추가
- UI에는 원문 텍스트 + 첨부 메타만 표시

## 3) 모델 목록
- main process에서 API 조회
- 5분 캐시
- 실패 시 fallback 모델 리스트 반환
- 렌더러 모델 선택 첫 옵션은 기본 모델(null)

## 4) 오류 처리
- stderr `error`/`fatal` 감지 시 이벤트 발행
- spawn 실패 시 오류 이벤트 발행
- UI 오류 배너 표시
- 오류 후에도 재전송 가능해야 함
